Restrição de Crédito - Método Penalidade¶

Considere o problema de maximização: $$ \max_{-\bar{A}\le s \le y_1}\left\{ u(\underbrace{y_1 - s}_{c_1}) - \beta u(\underbrace{y_2 - Rs}_{c_2}) \right\} $$ em que $$ u(c) = \frac{c^{1-\delta} - 1}{1 - \delta}. $$

Note que: \begin{align*} c_1 &= y_1 - s \ge 0\ \iff\ s \le y \\ c_2 &= y_2 + sR \ge 0\ \iff\ s \ge -\frac{y_2}{R}, \end{align*} então há uma imprecisão no problema de maximização quando $-\bar{A}\le s$, pois deveria ser $s \ge \max\left\{ -\frac{y_2}{R}; -\bar{A}\right\}$. No entanto, normalmente, $-\bar{A} \ge -\frac{y_2}{R}$ e, logo, usamos $-\bar{A}\le s$.

Seja o Lagrangeano dado por $$ L(s; \lambda) = u(y_1 - s) + \beta u(y_2 + Rs) + \lambda (s + \bar{A}) $$ em que o termo de restrição vem como soma ($+$), pois $(s + \bar{A}) \ge 0$. A derivada parcial em $s$ é dada por: $$ L_s (s, \lambda) = \beta R u'(y_2 + Rs) - u'(y_1 - s) + \lambda. $$

Quer-se computar $s^*$ tal que $$ L_s(s^*, \lambda) = 0 \qquad \text{e} \qquad \lambda(s^* + \bar{A}) = 0 $$

Observação 2. Note que $L_s(\underbrace{-y_2/R}_{s}; \lambda) = \infty > 0$ e $L_s(y_1; \lambda) = -\infty < 0$. Como $L_s(\cdot; \lambda)$ é contínua e estritamente decrescente,

  • para cada $\lambda \ge 0$, existe uma e somente uma raiz $s^*$ para $L_s(s;\lambda) = 0$
  • adicionalmente, $s^* \in (-y_2/R, y_1)$

Note que, podemos encontrar um $-y_2/R < s^* \le -\bar{A}$, o que não respeitaria a restrição e, portanto, precisamos de um programa que ajuste $L_s$ quando a solução $s^* < -\bar{A}$. Faremos isso, ajustando o valor do multiplicador de Lagrange, $\lambda$, que pode ser entendido como uma "penalização" na função objetivo pela violação da restrição $s^* + \bar{A} \neq 0$.

import numpy as np
import matplotlib.pyplot as plt

# Parâmetros
beta, delta = .5, .5
y1, y2, R, barA = 1, 2, 1.0, 0  # R é a taxa de juros

n_s = 30
s_grid = np.linspace(-y2/R, y1, n_s)

# Função Ls
def L_s(s, multiplier = 0.):  # Se multiplicador não for informado, será igual a 0 por padrão
    return beta*((y2 + R*s)**(-delta)) - (y1-s)**(-delta) + multiplier
    # Não se usa 'lambda' para o multiplicador de Lagrange, pois é utilizado para definição de funções no Python

# Função que retorna s*, dado multiplicador
def compute_s_star(multiplier):
    s_L, s_H = -y2/R, y1
    norm, tol = 1, 1e-6
    while norm > tol:
        s = (s_L + s_H) / 2
        CPOs = L_s(s, multiplier)
        
        if CPOs > 0:
            s_L = s
        else:
            s_H = s
        
        norm = abs(s_H - s_L)
    return s

Note que, embora $s^* = 0.9906 \ge 0 = -\bar{A}$, para o multiplicador de Lagrange, mu ($\lambda$), segue que que $$ \lambda (s^* + \bar{A}) = 9.906, $$ ou seja, viola a condição de slackness complementar ("errou a mão" na punição pela violação da restrição de crédito - escolha do multiplicador de Lagrange).

Agora, iremos fazer uma repetição para encontrar o multiplicador de Lagrange tal que $$ \lambda (s^* + \bar{A}) = 0. $$

  1. Escolhemos um multiplicador de Lagrange, mu ($\lambda$), estritamente positivo, calculamos o $s^*$ (dado $\lambda$) e encontramos a restrição de crédito.
  2. Caso a restrição de crédito $(s^* + \bar{A})$ seja positivo, diminuímos o valor do multiplicador. E, caso seja negativo, aumentamos o valor do multiplicador de Lagrange.
    • Normalmente, multiplica-se o multiplicador de Lagrange por $e^{(s^* + \bar{A})}$, pois
      • quando $(s^* + \bar{A}) > 0$, diminui o multiplicador
      • quando $(s^* + \bar{A}) < 0$, aumenta o multiplicador
      • quando $(s^* + \bar{A}) = 0$, mantém o valor do multiplicador
    • Outra forma de realizar isto é multiplicar por $1.1^{(s^* + \bar{A})}$: converge mais lentamente, mas é mais estável.
  3. Se $ \lambda (s^* + \bar{A}) = 0$, encontramos o $\lambda$ e o $s^*$ que resolvem o problema. Caso contrário, usa-se o novo multiplicador de Lagrange para executar o Passo 2 novamente.

# Executando as iterações para encontrar o mu (multiplicador de Lagrange)
norm_mu, tol_mu, mu = 1, 1e-6, 10
mu_grid, s_star_grid = [], []  # Listas para preencher nas iterações

while norm_mu > tol_mu:
    mu_grid.append(mu)
    s_star = compute_s_star(mu)
    s_star_grid.append(s_star)
    constraint = s_star + barA
    print(f"restrição = {round(constraint, 10)}, s* = {round(s_star, 6)}, e mu = {round(mu, 6)}")
    
    norm_mu = abs(mu * constraint)  # Não sabemos se mu ou a restrição vai para zero, só precisamos que a multiplicação vá
    
    # Não é necessário usar a estrutura de if, mas facilita o entendimento
    if constraint > 0:
        mu = np.exp(-constraint) * mu        
    else:
        mu *= np.exp(-constraint)  # Forma equivalente à de cima
        # mu *= 1.1 ** (-constraint)  # Outra forma mais "lenta" e estável de convergir para a solução
